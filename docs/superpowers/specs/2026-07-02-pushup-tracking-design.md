# Дизайн: трекинг отжиманий через камеру телефона

Дата: 2026-07-02
Статус: на ревью

## Контекст

push-ups-rpg — мобильное приложение (React Native), в котором игровой прогресс
завязан на реальные отжимания. Этот документ описывает модуль трекинга:
распознавание человека через фронтальную камеру и подсчёт повторений.
Игровая RPG-механика — вне скоупа этого документа.

Вводные, зафиксированные с пользователем:

- Платформа: React Native (iOS + Android).
- Телефон лежит/стоит на полу перед лицом пользователя, фронтальная камера
  смотрит на него снизу-спереди.
- Задача: определять руки, плечи и голову; подтверждать, что пользователь
  в упоре лёжа; считать повторение, когда руки сгибаются и разгибаются.
- Античита и соревновательного режима нет. Всё работает на устройстве,
  без сервера.

## Архитектура

Пайплайн из четырёх компонентов, каждый с одной ответственностью:

```
Камера → CameraSource → PoseEstimator → RepDetector → SessionController → события для игрового слоя
```

### CameraSource

- `react-native-vision-camera` v4, фронтальная камера.
- Запрашивает разрешение, управляет жизненным циклом камеры и frame
  processor (обработка кадров в worklet-потоке, UI не блокируется).

### PoseEstimator

- Модель MoveNet Lightning (single-pose) через `react-native-fast-tflite`
  + `vision-camera-resize-plugin` для даунскейла кадра под вход модели.
- Вход: кадр камеры. Выход: ключевые точки с confidence — нос, глаза, уши,
  плечи, локти, запястья (нижняя половина тела с этого ракурса не видна и
  не используется).
- Запасной вариант, если качество точек с нижнего ракурса окажется
  недостаточным: MediaPipe Pose Landmarker (`react-native-mediapipe`).
  Интерфейс «кадр → точки» позволяет заменить модель, не трогая остальное.

### RepDetector

Чистый TypeScript без нативных зависимостей. Вход — поток кадров с точками,
выход — события. Две обязанности:

**Контроль позиции.** Пользователь «в упоре лёжа», когда:

- плечи, локти и запястья видны с confidence выше порога;
- запястья в кадре ниже плеч;
- эта конфигурация стабильно держится ~1 секунду.

Пока позиция не подтверждена, счёт на паузе и наружу уходит `positionLost`
(UI показывает подсказку «займите упор лёжа»).

**Счёт повторов.** Угол в локте (плечо–локоть–запястье, берём средний по
двум рукам из тех, что видны) через конечный автомат:

```
UP (угол > ~155°) → DOWN (угол < ~95°) → UP  ⇒  repCounted
```

- Гистерезис между порогами защищает от дребезга на границе.
- Минимальная длительность полного повтора ~0.7 с защищает от двойного
  счёта на шуме.
- Все пороги (углы, длительности, confidence) — в одном конфиг-объекте;
  финальные значения подбираются на реальных тестах, поскольку нижний
  ракурс искажает видимые углы.

### SessionController

- Старт/стоп сессии, склейка компонентов.
- События наружу: `positionAcquired`, `positionLost`, `repCounted`,
  `sessionCompleted({ reps, durationSec })`.
- Игровой слой подписывается на события и ничего не знает про камеру
  и модель.

## UI и UX

Два экрана (MVP):

1. **Стартовый экран** — кнопка Start. По нажатию — запрос разрешения
   камеры (если ещё не выдано) и переход на экран тренировки.
2. **Экран тренировки** — превью фронтальной камеры на весь экран:
   - счётчик повторов сверху, крупно;
   - **скелет-оверлей**: поверх видео рисуются ключевые точки
     (голова, плечи, локти, запястья) и соединяющие их линии — скелет
     в реальном времени повторяет движения человека (референс —
     скриншот пользователя);
   - звуковой сигнал на каждый засчитанный повтор — лицо у пола,
     полагаться на звук удобнее, чем на экран;
   - подсказка на экране, пока позиция упора лёжа не распознана.

Отрисовка оверлея: `@shopify/react-native-skia` — canvas поверх превью
камеры, координаты точек передаются из frame processor через shared
values (Reanimated), чтобы не гонять их через JS-мост на каждом кадре.
Компонент `PoseOverlay` получает точки и ничего не знает про модель.

## Обработка ошибок

- Нет разрешения камеры → экран с объяснением и кнопкой запроса.
- Человек потерян в кадре / плохое освещение → пауза счёта, подсказка,
  счёт не сбрасывается.
- Модель не загрузилась / устройство не тянет → понятное сообщение об
  ошибке вместо падения.

## Тестирование

- `RepDetector` — чистые функции: юнит-тесты (Jest) на записанных
  JSON-фикстурах реальных сессий. Минимальный набор фикстур:
  правильные отжимания, неполная амплитуда, выход из кадра посреди
  подхода, дребезг на границе порога.
- Утилита записи фикстур: режим в приложении, который пишет поток точек
  в JSON-файл во время реальной сессии.
- `PoseEstimator`/`CameraSource` — ручная проверка на устройстве.

## Известный риск

С ракурса «телефон на полу перед лицом» локти при сгибании уходят вбок и
могут частично перекрываться телом — угол локтя может оказаться шумным.
Если подтвердится на тестах, добавляем вторичный сигнал: изменение
масштаба лица / межплечевого расстояния при приближении к камере
(при движении вниз лицо приближается и растёт в кадре). Архитектурно это
ещё один вход конечного автомата в `RepDetector`, замены компонентов не
требуется.

## Вне скоупа

- Античит и серверная верификация (решение пользователя: не нужно).
- Соревновательный режим.
- Контроль техники (глубина, прямая спина) сверх базового счёта.
- Игровая RPG-механика — отдельный слой, потребляющий события модуля.
