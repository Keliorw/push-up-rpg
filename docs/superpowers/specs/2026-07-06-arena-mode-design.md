# Design: режим «Арена» (забег на выживание)

**Дата:** 2026-07-06
**Область:** `web-game/` (+ чистая логика в `app/src/game/`)
**Статус:** утверждён к реализации
**Зависимость:** PR #11 (модалка рейтинга, `games/rating.png`) — ветка `web-arena`
застекана поверх `web-map-rating-modal`.

## 1. Цель

Арена — бесконечный забег: игрок отжиманиями убивает мобов с растущей
сложностью, пока не истечёт таймер текущего моба. Результат забега — число
убитых мобов; лучший результат сохраняется и участвует в отдельном
арена-рейтинге.

## 2. Игровой цикл

1. **Главное меню → «Arena»** → лобби арены (вместо прежнего полноэкранного
   XP-рейтинга — он удаляется; XP-рейтинг остаётся в модалке на карте, PR #11).
2. **Лобби** (`#screen-arena-lobby`): фон-колизей на весь экран; по центру
   кнопка **«Let's do it»** (рунная плита); под ней «Лучший результат: N»
   (скрыто при N=0); слева вверху кнопка «назад» в меню (стиль `#map-back`);
   справа вверху медальон рейтинга (стиль `#map-rating`) → модалка
   арена-рейтинга.
3. **«Let's do it»** → **превью первого моба** (переиспользуем DOM
   `#screen-card`: арт, имя, цель в HP и секундах; кнопки «В бой» / «Назад» в
   лобби — обработчики перекоммутируются на арену при входе).
4. **«В бой»** → экран загрузки (`ensureDetector`, как в кампании) → **бой** на
   DOM `#screen-workout`.
5. **Бой с мобом n**: обратный отсчёт `timer(n)`; каждый засчитанный повтор —
   удар (−1 HP, звук hit, `app.addRep()`); HP до нуля → моб убит
   (`kills += 1`), **отдых 30 с** (существующий оверлей `#wk-rest`: отсчёт, тап
   = пропустить) → следующий моб появляется **без превью** прямо в бою (смена
   карточки/имени/HP, таймер заново `timer(n+1)`).
6. **Таймер истёк** → забег окончен: камера останавливается, экран результата
   (`#screen-arena-result`): «Убито мобов: N», «Новый рекорд!» (если побит,
   звук victory) или «Рекорд: M»; кнопки **«Ещё раз»** (→ превью первого моба)
   и **«В лобби»**.
7. **Выход по ✕** во время боя = конец забега: тот же экран результата
   (результат фиксируется, XP синхронизируется).
8. Дневного лимита на арене **нет** (`dailyLock` не применяется). Отжимания
   арены засчитываются в XP (`totalReps`) как обычно.

## 3. Баланс (конфиг, легко крутить)

Все числа — в одном конфиг-объекте `ARENA_CONFIG` (`app/src/game/arena.ts`):

- `baseHp = 5`, `hpStep = 2` → `hp(n) = 5 + 2·(n−1)`; n — 1-based.
- `secondsPerRep = 4`, `baseTimerSec = 60` →
  **`timer(n) = max(baseTimerSec, hp(n) · secondsPerRep)`** — минимум 4 секунды
  на отжимание всегда.
- `restSec = 30` — отдых между мобами.

| Моб | HP | Таймер |
|----|----|----|
| 1 | 5 | 60с |
| 5 | 13 | 60с |
| 6 | 15 | 60с |
| 7 | 17 | 68с |
| 10 | 23 | 92с |
| 15 | 33 | 132с |

**Арт мобов:** монстры кампании по порядку `MONSTER_SEQUENCE`; моб n использует
монстра с индексом `(n−1) mod 40` (после 40-го — по кругу). Берутся только
имя/карточка; HP и таймер — аренные, из формул выше.

## 4. Данные

- **`bestArena: number`** — лучший результат (макс. убитых за один забег).
  - localStorage (`storage.ts`, рядом с `totalReps`), default 0.
  - Firestore `users/{uid}.bestArena`; **правила менять не нужно** (поле не
    валидируется, как `totalReps`).
  - `Profile` расширяется полем `bestArena`; `mergeProfile`:
    `bestArena = max(local, remote)` — не откатывается.
  - Запись: по концу забега (не на каждое убийство), вместе с `persistProfile`.
- **Арена-рейтинг:** `loadArenaLeaderboard(50)` в `remote-storage.ts`:
  `where('bestArena','>',0)` + `orderBy('bestArena','desc')` + `limit(50)`
  (одно поле — композитный индекс не нужен; у кого поля нет — не попадают).
  Строка: **ранг №, ник, убито мобов**; подсветка текущего игрока (uid).

## 5. UI / экраны

- **Лобби `#screen-arena-lobby`** — новый экран: фон `games/arena-bg.png`
  (как `#auth-bg`: absolute, cover); «Let's do it» — `.img-btn` с фоном
  `games/arena-start-button.png` и `<span>` поверх; «Лучший результат: N» под
  кнопкой; угловые кнопки как на карте (стили переиспользуются).
- **Модалка арена-рейтинга** — общий компонент с модалкой карты (PR #11):
  существующая разметка `#arena-modal` параметризуется заголовком и рендером
  строк; карта открывает её с XP-рейтингом, арена — с арена-рейтингом.
- **Превью моба** — DOM `#screen-card`; `card-target` показывает
  «HP × · время Y с»; свои обработчики кнопок на время арены.
- **Бой** — DOM `#screen-workout` с отличиями: `#wk-time` — обратный отсчёт
  (подсветка красным при <10с); `#wk-set` показывает «Убито: N»; остальной HUD
  (карточка моба, HP-бар, счётчик повторов, ✕) как есть.
- **Экран результата `#screen-arena-result`** — новый простой экран
  (в стиле `#screen-victory`).
- **Ассеты:** `~/Downloads/arena-bg.png` (768×1375) и
  `~/Downloads/arena-start-button.png` → ужать по ширине (~800px фон, ~600px
  кнопка, как остальные ассеты) → `web-game/games/`.

## 6. Модули и границы

- **`app/src/game/arena.ts` (новый, чистый TS, Jest):** `ARENA_CONFIG`,
  `mobHp(n)`, `mobTimerSec(n)`, `arenaMonster(n)` (индекс в
  `MONSTER_SEQUENCE`), state-машина забега:
  `{mobIndex, repsOnMob, kills, phase: 'fighting'|'resting'|'over'}` +
  `newRun()`, `onRep()`, `onRestOver()`, `onTimeout()`. Время инъектируется
  снаружи (тикает экран) — логика детерминирована и тестируема.
- **`web-game/src/battle-camera.ts` (новый):** извлечённый из
  `workout-screen.ts` цикл «камера → позы → RepDetector → onRep-колбэк +
  отрисовка скелета» (`startBattleCamera(video, canvas, detector, onRep) →
  {stop}`); pause-флаг для отдыха. Кампания и арена используют его.
- **`web-game/src/workout-screen.ts`:** рефактор на `battle-camera`; логика
  кампании (сеты, победа, dailyLock) не меняется.
- **`web-game/src/arena-lobby.ts` (новый):** лобби (рендер лучшего результата,
  проводка кнопок) + превью первого моба.
- **`web-game/src/arena-battle.ts` (новый):** бой арены поверх
  `#screen-workout`: обратный отсчёт, смена мобов, отдых, конец забега,
  сохранение результата, экран результата.
- **`web-game/src/sync.ts`:** `Profile.bestArena` + merge max (юнит-тест).
- **`web-game/src/storage.ts`:** `loadBestArena`/`saveBestArena`.
- **`web-game/src/remote-storage.ts`:** чтение/запись `bestArena`;
  `loadArenaLeaderboard`.
- **`web-game/src/main.ts`:** `btn-arena` → лобби; роутинг новых экранов;
  удаление прежнего полноэкранного `#screen-arena` и его проводки.
- **`web-game/index.html`:** новые экраны/стили; удаление `#screen-arena`.

## 7. Тестирование

- **Jest (`app/__tests__/arena.test.ts`):** формулы `mobHp`/`mobTimerSec`
  (граница 60с → рост: моб 6 = 60с, моб 7 = 68с); `arenaMonster` цикл после
  40-го; state-машина: повторы до убийства, отдых → следующий моб, таймаут →
  over, `onRep` в фазе resting/over — no-op.
- **node --test (`web-game/src/sync.test.ts`):** `mergeProfile` c `bestArena`
  (max, отсутствующее поле = 0).
- **E2E (браузер, localhost):** меню → лобби (фон, кнопки); Let's do it →
  превью → бой; убийство моба → отдых 30с → следующий моб без превью; таймаут
  → экран результата, рекорд сохранён (localStorage + Firestore); арена-рейтинг
  в модалке; ✕ фиксирует результат; кампания не сломана (бой по-прежнему
  работает).

## 8. Вне области (non-goals)

- RN-версия арены (веб-прототип, как весь текущий поток).
- Анти-чит (класс результата пишется клиентом, как `defeatedCount`/`totalReps`).
- Пагинация арена-рейтинга (топ-50), спец-арт мобов арены, награды/лут.
- Изменение баланса кампании.

## 9. Принятые решения

- Конец забега: **таймер на моба**; `timer(n) = max(60с, HP·4с)` — минимум 4
  секунды на отжимание, с ростом HP таймер растёт.
- Сложность: `hp(n) = 5 + 2·(n−1)`, отдых 30 с (тап = пропуск).
- Мобы: кампания по порядку, по кругу после 40-го.
- XP-рейтинг остаётся только на карте; на арене — свой рейтинг по `bestArena`.
- Выход ✕ = конец забега с фиксацией результата.
