# Design: рейтинг (Арена), экран загрузки уровня, размер камеры

**Дата:** 2026-07-03
**Область:** `web-game/` (статический сайт на GitHub Pages, аккаунты на Firebase)
**Статус:** утверждён к реализации
**Зависимость:** поверх фичи аккаунтов/облачного прогресса
(`2026-07-03-web-accounts-cloud-progress-design.md`, уже в `main`).

## 1. Проблемы / цели

Три независимых улучшения `web-game`:

1. **Рейтинг на Арене** — показать, «кто докуда дошёл» в кампании, с доп.
   метрикой XP (суммарные отжимания).
2. **Экран загрузки уровня** — предзагрузить модель MoveNet до показа камеры,
   чтобы пользователь не ждал загрузку с уже включённой камерой.
3. **Размер камеры в бою** — камера на всю высоту экрана и не шире 600px.

## 2. Функция 1: рейтинг (Арена) + XP

### Данные

Документ `users/{uid}` (Firestore) и локальный кэш расширяются полем:

- `totalReps: number` — XP, суммарное число засчитанных отжиманий за всё время.
  Монотонно растёт.

Итоговый документ: `{ nickname, defeatedCount, lastWorkoutDate, totalReps, updatedAt }`.

**Правила Firestore менять НЕ нужно:** чтение публичное (`allow read: if true`),
а `totalReps` в create/update не валидируется (существующие условия по
`nickname`/`defeatedCount` остаются в силе).

### Подсчёт и хранение XP

- В бою на каждое засчитанное отжимание (`repCounted`) инкрементируется
  локальный счётчик и немедленно пишется в localStorage (дёшево, оффлайн-safe).
- В Firestore `totalReps` синхронизируется **при выходе из боя** — и при победе
  над монстром, и при выходе по кнопке «✕» — одной записью на бой (не на реп).
- Правило слияния (`sync.ts` `mergeProgress`) расширяется:
  `totalReps = max(local.totalReps, remote.totalReps)` — не откатывается.

### Экран Арены

- Новый экран `screen-arena`; кнопка `btn-arena` в главном меню (сейчас
  неактивна) открывает его. На экране — кнопка «назад» в меню.
- Запрос: `getDocs(query(collection(db,'users'), orderBy('defeatedCount','desc'),
  limit(50)))`. Затем **клиентская до-сортировка** по `totalReps` desc внутри
  равного `defeatedCount` (композитный индекс Firestore не нужен).
- Строка рейтинга: **ранг №, ник, локация, XP**.
  - «Локация» выводится из `defeatedCount` тем же способом, что в `map.ts`
    (`currentLocationIndex`): по `id` текущего монстра `locN-…`. Показываем номер
    и название локации; если кампания пройдена (`defeatedCount` = длина
    последовательности) — «Пройдено».
  - «XP» = `totalReps`.
- Строка текущего игрока **подсвечена** (по совпадению uid).

## 3. Функция 2: экран загрузки уровня (предзагрузка MoveNet)

### Текущее поведение (проблема)

В `workout-screen.ts` `run()` порядок: запросить камеру → `video.play()` →
`tf.setBackend('webgl')` + `tf.ready()` + `poseDetection.createDetector(...)` (это
скачивание весов модели) → цикл. То есть модель грузится уже при включённой
камере, и детектор создаётся заново на каждый бой.

### Новое поведение

- **Кэш детектора на уровне модуля**: `ensureDetector(): Promise<Detector>` —
  инициализирует бэкенд tf и создаёт MoveNet один раз, далее возвращает
  закэшированный экземпляр (общий на все бои). Промис тоже кэшируется, чтобы
  параллельные вызовы не создавали две модели.
- **Фоновая предзагрузка**: `ensureDetector()` запускается в фоне сразу после
  входа пользователя (не блокирует меню).
- **Экран загрузки** `screen-loading` (спиннер + «Загрузка уровня…»): при
  «Начать тренировку» показываем его и ждём `ensureDetector()`. Когда готов —
  включаем камеру и запускаем бой. Если детектор уже закэширован — промис
  разрешается мгновенно, экран практически не задерживает.
- **Рефактор `workout-screen.ts`**: логика загрузки модели выносится в
  `ensureDetector()`; `startWorkout(app, detector)` получает готовый детектор и
  занимается только камерой, HUD и циклом распознавания. Ошибка загрузки модели
  показывается на экране загрузки с возможностью вернуться в меню.

## 4. Функция 3: камера на всю высоту, max 600px ширины

- `#screen-workout` при активности — полноэкранный слой, вне ограничения `#app`
  (max 480px): `position: fixed; inset: 0`.
- `#wk-stage`: `height: 100dvh; width: min(100vw, 600px); margin: 0 auto`
  (вместо `aspect-ratio: 3/4; width: 100%`). Существующий `transform: scaleX(-1)`
  (зеркало) сохраняется.
- `#wk-video`, `#wk-overlay` — `object-fit: cover` (уже так); скелет-оверлей
  рисуется в пиксельных координатах видео и масштабируется тем же `cover`, потому
  остаётся выровненным.
- HUD-элементы позиционированы абсолютно от `#wk-stage`; при более высоком кадре
  остаются видимыми у верхней части. При необходимости отступы подстраиваются
  (мелкая доводка, не блокер).

## 5. Модули и границы

- `web-game/src/sync.ts` — расширить `mergeProgress` полем `totalReps` (чистая
  логика, юнит-тест).
- `web-game/src/levels.ts` (новый) — чистая функция
  `locationLabel(defeatedCount): { index: number|null, name: string }` для
  рейтинга и, при желании, переиспользования в `map.ts` (юнит-тест).
- `web-game/src/storage.ts` / профиль — хранить `totalReps` рядом с
  `Progression` в localStorage.
- `web-game/src/remote-storage.ts` — писать/читать `totalReps`; новый
  `loadLeaderboard(limit): Promise<Array<{uid, nickname, defeatedCount, totalReps}>>`.
- `web-game/src/arena-screen.ts` (новый) — рендер таблицы рейтинга.
- `web-game/src/pose-model.ts` (новый) — `ensureDetector()` с кэшем.
- `web-game/src/workout-screen.ts` — рефактор: принимает готовый детектор.
- `web-game/src/main.ts` — фоновая предзагрузка после входа; роутинг
  `screen-loading`, `screen-arena`; учёт `totalReps` при сохранении.
- `web-game/index.html` — экраны `screen-loading`, `screen-arena`; стили камеры.

## 6. Тестирование

- **Юнит:** расширенный `mergeProgress` (в т.ч. `totalReps = max`);
  `locationLabel` (границы: 0 → локация 1; середина; пройдено).
- **E2E (браузер, localhost):** рейтинг показывает игроков и подсвечивает
  текущего; модель грузится на экране загрузки до камеры (в консоли нет
  «Загружаю модель…» уже при видео); повторный бой — без ожидания (кэш); камера
  ≤600px шириной и на всю высоту; XP растёт и попадает в Firestore.

## 7. Вне области (non-goals)

- Серверный анти-чит (XP пишется клиентом — как и `defeatedCount`).
- Композитные индексы Firestore (используем одиночный `orderBy` + клиентскую
  до-сортировку).
- Пагинация рейтинга (фиксированный топ-50).
- Изменение общего игрового ядра `app/src/game/*` (RN-приложение не трогаем;
  `totalReps` — поле веб-профиля, вне общего типа `Progression`).

## 8. Принятые решения

- Рейтинг: локация (из `defeatedCount`) + **XP = суммарные отжимания**; сортировка
  по `defeatedCount`, при равенстве — по `totalReps`; топ-50; подсветка текущего.
- Экран загрузки уровня + фоновая предзагрузка + кэш детектора.
- Камера: `height: 100dvh`, `width: min(100vw, 600px)`.
