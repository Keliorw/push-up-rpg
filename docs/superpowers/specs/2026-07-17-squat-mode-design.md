# Режим приседаний: выбор упражнения перед боем + детектор приседаний

Дата: 2026-07-17. Статус: одобрено (цель приседаний ×1.5 от отжиманий).

## Зачем

Второй способ пройти монстра: перед схваткой игрок выбирает «Отжимания» или
«Приседания». Приседания легче, поэтому монстр требует их больше: ×1.5 от
числа отжиманий, округление вверх (10 отжиманий → 15 приседаний).

## Части

### 1. SquatDetector (app/src/pose/)

Отдельный класс рядом с `RepDetector` — откалиброванный детектор отжиманий не
трогаем. Тот же контракт: `process(pose, tMs): DetectorEvent[]` с событиями
`positionAcquired | positionLost | repCounted` и объектом `debug` с живыми
сигналами. Пороги — в отдельном `DEFAULT_SQUAT_CONFIG` (`squat-config.ts`).

Гейт «стою в полный рост»: видны плечи+бёдра+колени, вертикальный порядок
(плечи выше бёдер, бёдра не ниже колен сверх допуска). Допуск нужен, чтобы гейт
не рвался в нижней точке приседа (присед длится дольше grace-окна).

Повтор (вверх→вниз→вверх), «вниз» — любой из двух сигналов:
- **просадка таза** (основной, устойчив для фронтальной камеры): бёдра
  опустились относительно верхней базовой линии на долю длины торса;
- **угол колена** (бедро–колено–лодыжка; хорош для вида сбоку): согнут ≤
  порога, разогнут ≥ порога (гистерезис).

Точные пороги — черновые до калибровки по видео пользователя.

### 2. Тестовая страничка (web-demo/squat.html)

Как существующая демка отжиманий, плюс **загрузка видеофайла**: источник —
камера ИЛИ файл (пользователь записывает видео приседаний, файл прогоняется
через детектор многократно при калибровке, с перемоткой). На экране: скелет,
счётчик, статус гейта, фаза, угол колена, просадка таза, min/max диапазоны,
пороги; кнопка сброса; выгрузка лога сигналов в JSON.

Сборка: `npx esbuild web-demo/src/squat-demo.ts --bundle --format=esm
--outfile=web-demo/squat.js` (как у demo.js).

### 3. Выбор режима перед боем (web-game)

На карточке монстра перед стартом — выбор «Отжимания / Приседания».
`startWorkout` и `startBattleCamera` получают режим: он определяет детектор
(RepDetector / SquatDetector), подсказку («Займи упор лёжа» / «Встань в полный
рост») и цель повторов (для приседаний `ceil(repsPerSet × 1.5)`).

## Порядок работ

1. SquatDetector + тесты (jest, как RepDetector.test.ts) + страничка. 
2. Пользователь записывает видео → калибровка порогов по видео.
3. Интеграция выбора режима в игру (после калибровки).

## Тестирование

- Юнит-тесты FSM детектора на синтетических позах (стоя/присед, фантомы:
  наклон корпуса без приседа не считается — таз не проседает).
- Калибровка по реальному видео через страничку.
- Смоук страницы: загрузка без камеры не падает, файл-режим работает.
