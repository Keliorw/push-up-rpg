# web-demo — проверка детекции через камеру компьютера

Небольшая **браузерная** демка, чтобы увидеть, как работает распознавание позы и
подсчёт отжиманий, не собирая мобильное приложение и без Android-тулчейна.

> Это **не** боевое приложение. Мобильный экран использует нативную камеру
> (react-native-vision-camera) и tflite-модель — на десктопе они не работают.
> Демка переиспользует **тот же самый** `RepDetector` (и геометрию/конфиг) из
> `../app/src/pose/`, а вместо нативной модели берёт keypoints из TensorFlow.js
> MoveNet, вместо Skia рисует на `<canvas>`. То есть проверяет реальную логику
> счёта — только источник точек и отрисовка другие.

## Запуск (нужен интернет для CDN TF.js и веб-камера)

Камера в браузере доступна только на `localhost` или `https`, поэтому файл нельзя
открыть напрямую (`file://`) — надо поднять локальный сервер из папки `web-demo`:

```bash
cd web-demo
python3 -m http.server 8080
# или: npx --yes serve -l 8080 .
```

Затем открыть в браузере (Chrome/Edge надёжнее всего):

```
http://localhost:8080/
```

Разрешить доступ к камере. Через пару секунд появится скелет поверх видео и
счётчик сверху.

## Как это работает (та же логика, что на телефоне)

**Гейт позиции** — «горизонтальное положение»:
- Если камера видит корпус и ноги (плечи, бёдра, колени) — строгий гейт **планки**:
  корпус должен быть прямым (угол плечо–бедро–колено ≥ `plankBodyMinAngleDeg`).
  Сидя на стуле бедро согнуто → позиция не подтверждается (это и была причина, по
  которой «сидя и сгибая руки» раньше засчитывалось).
- Если ног в кадре нет — **запасной** гейт по рукам (кисти ниже плеч).

**Счёт повтора** — срабатывает ЛЮБОЙ из двух сигналов:
1. **Вертикальное проседание корпуса** (основной, устойчив для фронтального
   ракурса): плечи опускаются относительно «верхней» точки на долю длины торса
   (`descentDownFrac`), затем возвращаются. Именно это чинит «10 отжиманий → 1».
2. **Угол локтя** (запасной): < 95° → > 155° (хорош для вида сбоку/сидя).

**Отладочная строка внизу** показывает живые сигналы детектора: `гейт`,
`в позиции`, `фаза` (up/down), `корпус` (угол планки), `проседание` (текущее
значение и порог), `локоть`. По ней калибруйте пороги: сделайте несколько
настоящих отжиманий и посмотрите, до каких значений доходит `проседание` в
нижней точке — если оно не дотягивает до порога, уменьшите `descentDownFrac`
в `../app/src/pose/config.ts` и пересоберите демо.

Сидя за ноутбуком настоящие отжимания не сделать. Чтобы камера видела ноги для
гейта планки, поставь ноутбук пониже/подальше. Если ноги не влезают — сработает
запасной гейт по рукам, и счёт пойдёт по локтю (как раньше).

## Пересобрать после правок логики

`demo.js` собран из `src/demo.ts` (который импортирует настоящий `RepDetector`).
После изменений в `../app/src/pose/*` или в `src/demo.ts` пересоберите:

```bash
cd ..   # в корень репозитория push-up-rpg
npx --yes esbuild web-demo/src/demo.ts --bundle --format=esm --outfile=web-demo/demo.js
```

## Ограничения демки

- MoveNet в TF.js — не та же сборка, что tflite на телефоне, но семейство и порядок
  точек те же (0=нос … 10=правое запястье), поэтому счёт ведёт себя эквивалентно.
- Пороги в `../app/src/pose/config.ts` подобраны под ракурс «телефон на полу»;
  для настольной камеры их, возможно, придётся подкрутить — правьте `DEFAULT_CONFIG`
  и пересоберите.
